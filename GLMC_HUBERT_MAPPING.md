The rationale is that the residual stream carries the richest symmetry structure, while heads and neurons have discrete (permutation) symmetries.\n\n## 1.3 How Transforms Are Applied Per Component\n\nFrom `ortho_residual()` in `weight_matching.py`, the global O is applied as follows (all operations on model B):\n\n| Component | Transform | Mathematical Operation |\n|---|---|---|\n| Token embeddings (wte) | Right-multiply by O | `wte = wte @ O` |\n| Position embeddings (wpe) | Right-multiply by O | `wpe = wpe @ O` |\n| LayerNorm bias (ln_1, ln_2) | Left-multiply by O^T | `bias = O^T @ bias` |\n| Attention input (c_attn) | Left-multiply by O^T | `W = O^T @ W` |\n| Attention output (c_proj) weight | Right-multiply by O | `W = W @ O` |\n| Attention output (c_proj) bias | Right-multiply by O | `bias = bias @ O` |\n| FFN input (c_fc) weight | Left-multiply by O^T | `W = O^T @ W` |\n| FFN output (c_proj) weight | Right-multiply by O | `W = W @ O` |\n| FFN output (c_proj) bias | Right-multiply by O | `bias = bias @ O` |\n| Final LayerNorm bias | Left-multiply by O^T | `bias = O^T @ bias` |\n| LM head | Right-multiply by O | `W = W @ O` |\n\n**Note on LayerNorm**: GLMC first absorbs LayerNorm scale into adjacent weights (making weight=1), then replaces LayerNorm with RMSNorm. After this, only the bias remains, and the bias transforms as `O^T @ bias` because it lives in the residual stream.\n\n## 1.4 Attention Head Handling\n\nHeads are handled through a **two-phase** approach:\n\n**Phase 1: Attention circuit projection** (`project_to_attn_circuits`). Before alignment, GLMC reparameterizes Q/K/V/O into \"attention circuits\":\n- Computes `QK_h = Q_h^T @ K_h` per head (the attention pattern circuit)\n- Computes `OV_h = O_h^T @ V_h` per head (the value circuit)\n- Then \"splits\" these back into new Q, K, V, O matrices (using identity as the K/V factor)\n\nThis reparameterization ensures that per-head rotations within Q and K that cancel in `Q^T K` are factored out, making the head representation canonical.\n\n**Phase 2: Head permutation** (`permute_heads`). After residual alignment, the cost matrix for matching head i of model A to head j of model B is:\n```\ncost[i,j] = ||QK0[i] - QK1[j]||_F + ||OV0[i] - OV1[j]||_F\n```\nSolved via optimal transport to get permutation P_heads. Then Q, K, V weight rows are permuted in blocks of `d_head`, and O weight columns are permuted in blocks of `d_head`.\n\n**Per-head transforms are permutations only, not orthogonal.** There is no per-head orthogonal alignment in GLMC.\n\n## 1.5 Residual Connection Handling\n\nThe consistency constraint is: every path through the residual stream must see a consistent coordinate system. GLMC enforces this by:\n\n1. Applying O to every \"entry point\" into the residual stream (embeddings, c_proj output, mlp_proj output)\n2. Applying O^T to every \"exit point\" from the residual stream (c_attn input, c_fc input, LN bias, lm_head)\n3. The identity skip connection automatically stays consistent because `x_new = x @ O` flows through unchanged\n\nAdditionally, `apply_mean_subtraction_to_weights` projects out the mean from all residual-stream outputs (via `M = I - 11^T/d`). This removes a degree of freedom (the mean direction) that RMSNorm is invariant to, making the alignment problem cleaner.\n\n## 1.6 Graph Traversal Order\n\nThe algorithm in `weight_matching` proceeds:\n\n1. **Pre-processing** (in `GPTMerger.__init__`):\n   - `absorb_ln_scale` -- fold LN gamma into adjacent weights\n   - `replace_layernorm` -- swap LayerNorm for RMSNorm\n   - `apply_mean_subtraction_to_weights` -- project out mean from residual outputs\n   - `project_to_attn_circuits` -- reparameterize attention into canonical circuits\n\n2. **Iterative residual alignment** (15 iterations):\n   - Gather all residual-dimension weight slices from both models\n   - Solve Procrustes for global O\n   - Apply O to all of model B's weights via `ortho_residual`\n\n3. **Per-layer discrete alignment** (single pass, layer 0 to 11):\n   - For each layer: compute head permutation P_heads, apply it\n   - For each layer: compute FFN permutation P_mlp, apply it\n\n4. **Merge**: Interpolate between aligned model B and model A with coefficient lambda\n\n---\n\n# TASK 2: Mapping to HuBERT/MERT Architecture\n\n## 2.0 Critical Architecture Observation: Post-LN, Not Pre-LN\n\nThe actual HuBERT forward pass is:\n\n```\n# ATTENTION BLOCK\nattn_residual = hidden_states                    # save residual\nhidden_states = attention(hidden_states)          # attention on RAW input (no pre-norm!)\nhidden_states = attn_residual + hidden_states     # residual add\nhidden_states = layer_norm(hidden_states)         # POST-norm\n\n# FFN BLOCK\nffn_residual = hidden_states                      # save residual (post-normed)\nhidden_states = feed_forward(hidden_states)       # FFN on post-normed input\nhidden_states = ffn_residual + hidden_states      # residual add\nhidden_states = final_layer_norm(hidden_states)   # POST-norm\n```\n\nThis is **post-LN** architecture (norm AFTER residual addition).
